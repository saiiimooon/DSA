Understanding the AVL Tree

An AVL Tree (named after Adelson-Velsky and Landis) solves the biggest problem of a standard Binary Search Tree: skewing. It enforces a strict rule to ensure search times always remain lightning-fast.

The Balance Factor (BF)

Every node calculates its tilt:
BF = Height(Left) - Height(Right)

A node is balanced only if its BF is -1, 0, or 1.

The Rule of One

If any node's Balance Factor hits 2 or -2, the tree is officially imbalanced. The algorithm immediately pauses to perform a Rotation to level it out.

Java Implementation: The Node & Rotation

To code this, we create a Node class that tracks height, and a method that physically swaps the pointers during a rotation.

class Node {
    int key, height;
    Node left, right;
    
    Node(int d) {
        key = d;
        height = 1;
    }
}

// Right Rotation (LL Case Fix)
Node rightRotate(Node y) {
    Node x = y.left;
    Node T2 = x.right;

    // Perform rotation
    x.right = y;
    y.left = T2;

    // Update heights
    y.height = Math.max(height(y.left), height(y.right)) + 1;
    x.height = Math.max(height(x.left), height(x.right)) + 1;

    return x; // Return new root
}

Java Implementation: Insertion Logic

Every time we insert, we update the height, check the Balance Factor, and trigger the correct rotation if necessary.

Node insert(Node node, int key) {
    /* 1. Normal BST insertion logic goes here */

    /* 2. Update height & Get Balance Factor */
    node.height = 1 + Math.max(height(node.left), height(node.right));
    int balance = getBalance(node);

    // LL Case
    if (balance > 1 && key < node.left.key) 
        return rightRotate(node);

    // LR Case
    if (balance > 1 && key > node.left.key) {
        node.left = leftRotate(node.left);
        return rightRotate(node);
    }
    return node;
}

Real-Life Applications

Why do we use AVL trees in the real world? Because they guarantee an extremely fast O(log n) search time, regardless of how data is inserted.

  • 🗄️ Databases: Used in internal systems and indexes where fast lookups are critical. A balanced tree ensures no search takes longer than necessary.
  • 🔍 Search Engines: Essential for maintaining sorted, easily accessible dictionaries of keywords and web indexing where millions of fast queries happen per second.
  • 💻 Large-Scale Memory Systems: Used by operating systems to keep track of available memory blocks perfectly balanced and sorted by size.
